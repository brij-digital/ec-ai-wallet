Runtime Spec V1 Reference

Program-specific runtime logic in AppPack

This page documents the current solana-agent-runtime.v1 contract and the deterministic runtime layer that sits on top of Codama.

Scope boundary

Codama
  • instruction shape
  • named accounts
  • signers
  • fixed/default accounts
  • PDA-backed defaults when declared
Runtime spec
  • views
  • writes
  • reusable transforms
  • ordered runtime steps
  • small transaction-envelope logic

Shared operation core

AttributeRequiredTypeMeaning
stepsNoarrayOrdered array of operationStepSpec.

Each operation step is either a direct runtime step such as wallet_pubkey, decode_account, decode_accounts, ata, or pda, or a transform fragment invocation like { "kind": "transform", "transform": "quote_exact_in__quote_math" }.

Write inputs are not declared explicitly in the runtime file. They come from the referenced Codama instruction, narrowed to the Codama args/accounts still referenced through $input.* inside the write.

transforms

Top-level transforms is the only place where reusable transform fragments are declared. Operations reference these fragments from ordered steps. 

{
  "transforms": {
    "swap_direction": [
      {
        "name": "a_to_b",
        "kind": "compare.equals",
        "left": "$whirlpool_data.token_mint_a",
        "right": "$input.token_in_mint"
      }
    ]
  },
  "views": {
    "quote_exact_in": {
      "steps": [
        { "kind": "transform", "transform": "swap_direction" }
      ]
    }
  }
}

The JSON schema is the source of truth for the full step catalog and exact attributes.

Scoped collection transforms

list.map, list.flat_map, and list.reduce can run nested steps inside the collection scope. 

{
  "name": "quote_row",
  "kind": "transform",
  "transform": "quote_tick",
  "bindings": {
    "tick": "$item",
    "direction": "$a_to_b"
  },
  "output": "$payload"
}

This is one of the main runtime improvements exercised by the Orca harness.

Concrete Orca example

The current Orca runtime pack is a good boundary test for this spec. It exercises exact-input quotes, exact-output quotes, ordered steps, scoped collection transforms, Codama-backed writes, and preview envelope instructions. 

{
  "protocol_id": "orca-whirlpool-mainnet",
  "program_id": "whirLbMiicVdio4qvUfM5KAg6Ct8VwpYzGff3uctyCc",
  "codama_path": "/idl/orca_whirlpool.codama.json",
  "transforms": {
    "quote_exact_in__derive_tick_arrays": [],
    "quote_exact_in__quote_math": []
  },
  "views": {
    "quote_exact_in": {
      "load_instruction": "swap_v2",
      "inputs": {
        "token_in_mint": "token_mint",
        "token_out_mint": "token_mint",
        "amount_in": "u64",
        "slippage_bps": "u16",
        "whirlpool": "pubkey",
        "unwrap_sol_output": "bool"
      },
      "load_instruction_bindings": {
        "args": {
          "amount": "$input.amount_in"
        },
        "accounts": {
          "whirlpool": "$input.whirlpool"
        }
      },
      "steps": [
        { "name": "wallet", "kind": "wallet_pubkey" },
        {
          "name": "whirlpool_data",
          "kind": "decode_account",
          "address": "$input.whirlpool",
          "account_type": "Whirlpool"
        },
        { "kind": "transform", "transform": "quote_exact_in__derive_tick_arrays" },
        {
          "name": "tick_arrays_data",
          "kind": "decode_accounts",
          "addresses": "$tick_arrays",
          "account_type": "TickArray"
        },
        { "kind": "transform", "transform": "quote_exact_in__quote_math" }
      ],
      "pre": [
        {
          "kind": "spl_ata_create_idempotent",
          "payer": "$wallet",
          "ata": "$instruction_accounts.token_owner_account_a",
          "owner": "$wallet",
          "mint": "$whirlpool_data.token_mint_a"
        }
      ],
      "output": {
        "type": "object",
        "source": "$derived",
        "object_schema": {
          "entity_type": "orca_quote_exact_in",
          "identity_fields": ["whirlpool"],
          "fields": {
            "whirlpool": { "type": "pubkey" },
            "token_in_mint": { "type": "pubkey" },
            "token_out_mint": { "type": "pubkey" },
            "amount_in": { "type": "u64" },
            "slippage_bps": { "type": "u16" },
            "estimated_out": { "type": "u64" },
            "minimum_out": { "type": "u64" },
            "a_to_b": { "type": "bool" },
            "pool_fee_bps": { "type": "number" }
          }
        }
      }
    }
  },
  "writes": {
    "swap_exact_in": {
      "instruction": "swap_v2",
      "args": {
        "amount": "$input.amount",
        "other_amount_threshold": "$input.other_amount_threshold",
        "sqrt_price_limit": "$input.sqrt_price_limit",
        "amount_specified_is_input": "$input.amount_specified_is_input",
        "a_to_b": "$input.a_to_b",
        "remaining_accounts_info": null
      },
      "accounts": {
        "whirlpool": "$input.whirlpool",
        "tick_array0": "$input.tick_array0",
        "tick_array1": "$input.tick_array1",
        "tick_array2": "$input.tick_array2"
      }
    }
  }
}
