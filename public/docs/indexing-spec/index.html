Indexing Spec V1 Reference

Declarative decoder runtime and indexed read surfaces

This page documents the current declarative-decoder-runtime.v1 contract. It covers ingest sources, match rules, pipelines, projections, and operations.*.index_view.

Scope boundary

Indexing spec owns
  • decoder artifacts
  • ingest source definitions
  • match and routing rules
  • canonical record/state pipelines
  • projection definitions
  • indexed read and discovery views
Indexing spec does not own
  • Codama instruction execution
  • runtime write preparation
  • wallet transaction submission
  • runner flow orchestration

emitSpec

emit can contain record, state, or both. Both use the same core fields: recordKind, recordName, subjectId, optional resourceId, optional signature, optional slot, optional eventTime, and payload. record also supports optional participantId.

projectionSourceSpec

Main attributes: canonicalSource, protocolId, optional recordKind, optional recordName, optional seriesId, optional subjectInput, optional resourceInput, and optional timeField.

projectionAggregationSpec

Required attributes: name, field, fn. Supported functions: first, last, min, max, sum, avg, count, count_distinct.

operations.*.index_view

This is the part of the indexing spec that now owns indexed read and discovery surfaces.

canonicalViewSpec

Main attributes: source, protocol_id, optional record_kind, optional record_name, optional series_id, optional subject_input, optional resource_input, optional subject_field, optional resource_field, optional time_field.

viewProjectionSpec

Main attributes: projection_kind, optional source_operation, optional bucket, optional group_by, optional aggregations, optional sort_by.

Rule: if source_kind = account_changes, then bootstrap and query are required. Otherwise, canonical and projection are required.

Current required field is only kind = account. It also allows optional source_kind, freshness, entity_type, title, description, bootstrap, refresh, query, target, and select.

Pump AMM example

{
  "sources": {
    "pump_swap_crawler": {
      "kind": "rpc_transaction_crawler",
      "decoderRef": "pump_swap",
      "decodeTargets": ["instruction", "log_event"]
    }
  },
  "matchRules": [
    {
      "source": "pump_swap_crawler",
      "match": { "decodedType": "event", "name": "BuyEvent" },
      "pipeline": "trade_buy"
    }
  ],
  "pipelines": {
    "trade_buy": {
      "resolve": [
        {
          "name": "poolMeta",
          "resolve": "state.lookup",
          "recordName": "PoolMetadata",
          "subjectId": "$decoded.pool",
          "required": true
        }
      ],
      "compute": [
        {
          "name": "priceQuote",
          "compute": "math.safe_div",
          "dividend": "$quoteAmountUi",
          "divisor": "$baseAmountUi"
        }
      ],
      "emit": {
        "record": {
          "recordKind": "event",
          "recordName": "BuyEvent",
          "subjectId": "$ctx.mint",
          "resourceId": "$ctx.pool",
          "payload": {}
        }
      }
    }
  }
}
